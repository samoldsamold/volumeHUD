# Tests

Use this directory for automated tests, test helpers, documented verification entry points, or explicit test gaps.

Every spec must name the test commands that prove its acceptance criteria. Every task must run those commands before completion and record the actual output.

If a task has no automated test, record the objective verification command that was run, such as a formatter check, linter, docs build, `git diff --check`, or a project-specific smoke test, and record the remaining test gap. Verification can be narrow, but it is never optional.
